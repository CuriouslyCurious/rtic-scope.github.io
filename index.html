1. About

RTIC Scope is a zero-overhead framework for recording and analyzing execution traces from RTIC applications on ARM Cortex-M targets.

1.2. Project repositories/crates

frontend-dummy
A "noop" frontend implementation that writes received api::EventChunk structs to stderr with nanosecond timestamp information.
cargo-rtic-scope
The RTIC Scope backend which builds the target application, recovers trace information, traces the target, replays traces, etc.
api
The common API used between the RTIC Scope backend and all frontends.
examples
A set of example target applications where the RTIC Scope framework is applied. These are documented below in 4.
rtic-trace
The no_std crate used to configure the target for tracing purposes.
itm-decode
A host-side library that decodes the binary trace stream received from the target to workable Rust structures.
rfcs
A catch-all meta-repository for discussions that encompasses more than a single repository.
rtic-scope.github.io
The source code for this page.

2. Requirements

2.1. Hardware

  • An embedded system with an ARM Cortex-M MCU and available asyncronous trace pin output.
  • Serial hardware.

2.2. Software

  • A recent Linux-based operating system.

5. How it works

cargo-rtic-scope is abstracted in a single-producer, multiple-consumer fashion, where … the heavy work is done while tracing:

  • build the target application
  • generate task resolve maps from application RTIC #[app] declaration
  • reset the target and sample timestamp
